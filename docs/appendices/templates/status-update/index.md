# 状況共有テンプレ

## 更新頻度（目安）

| Severity | 更新頻度（目安） | 補足 |
|---|---|---|
| P0 | 15 分ごと | 影響/復旧見込みが変わらなくても「状況維持」を共有する |
| P1 | 30〜60 分ごと | 主要な切り分け結果・次アクションを更新する |
| P2 | 数時間ごと | 影響範囲/暫定対応/恒久対応の方針を更新する |
| P3 | 定期見直し（例: 週次） | バックログ管理し、放置ではなく管理する |

## 更新に含める最小項目

- 影響（誰に/何が/どの程度）
- 次アクション（担当/期限）
- 次回更新予定時刻（次にいつ共有するか）

## 記入例（最小）

```md
## いま何が起きているか
- 決済 API で 5xx が増加

## 何を試したか
- 直近デプロイ差分を確認→変更Aが疑わしい

## 次に何をするか
- ロールバック可否の確認→実施判断（検証指標: エラー率/レイテンシ）

## 影響見込み
- 決済失敗が一部顧客で継続（失敗率 6%、暫定）

## 次回更新予定時刻
- 11:00 JST
```
