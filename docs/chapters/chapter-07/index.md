# 第7章：復旧とロールバック判断（暫定/恒久、リスク管理）

## この章で学ぶこと

- 暫定 vs 恒久を分けて考える
- ロールバック判断の基準を持つ
- 復旧によるリスク（副作用）を評価する

## 成果物（または判断基準）

- 復旧方針（暫定/恒久）と判断根拠
- ロールバック手順と検証項目

## 本文

復旧判断は“正しさ”ではなく“リスク最小化”で行う。暫定対応で止血し、恒久対応は落ち着いて行う。

### 判断の観点

- 影響（継続時間と範囲）
- ロールバック可否
- 副作用（データ整合性、再発）
- 恒久対応の見通し

## 具体例（場当たり→再現性）

### 悪い例（場当たり）

```md
原因が分かるまで何もしない
ロールバック手順が無い
```

### 良い例（再現性）

```md
暫定: 機能フラグOFFで影響を止める
恒久: 原因特定後に修正PR
ロールバック: 手順/検証をRunbookへ記録
判断根拠: 影響と副作用を比較
```

## チェックリスト

- [ ] 暫定/恒久が分離されている
- [ ] ロールバック可否が評価されている
- [ ] 検証項目がある

## まとめ

- 調査は“仮説→検証→記録”をセットで回す
- インシデント対応は“技術”だけでなく“運用（連絡/判断/役割）”が重要
